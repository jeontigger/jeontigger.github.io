## 📝 모범 답변

### ✅ 기본 개념

1. 여러 파일 디스크립터의 I/O 준비 상태를 확인하는 함수입니다.
2. select와 유사하지만 더 유연한 인터페이스를 제공하는 함수입니다.
3. 여러 소켓을 단일 스레드에서 효율적으로 처리하기 위해서입니다.

### ✅ select() 동작 원리

1. 지정된 파일 디스크립터들이 읽기/쓰기/예외 상태가 준비될 때까지 대기합니다.
2. 최대 파일 디스크립터 번호, 읽기/쓰기/예외 fd_set, 타임아웃입니다.
3. 준비된 파일 디스크립터의 개수, 0(타임아웃), -1(에러)를 반환합니다.
4. 파일 디스크립터 집합을 나타내는 비트마스크 구조체입니다.

### ✅ poll() 동작 원리

1. pollfd 배열의 각 파일 디스크립터의 이벤트를 확인합니다.
2. fd(파일 디스크립터), events(관찰할 이벤트), revents(발생한 이벤트)입니다.
3. POLLIN은 읽기 가능, POLLOUT은 쓰기 가능, POLLERR는 에러를 의미합니다.

### ✅ select vs poll 차이점

1. select는 fd_set을 사용하고, poll은 pollfd 배열을 사용합니다.
2. FD_SETSIZE(보통 1024)로 제한됩니다.
3. 파일 디스크립터 개수 제한이 없고, 더 유연한 이벤트 처리가 가능합니다.
4. 더 널리 지원되고, 크로스 플랫폼 호환성이 좋습니다.

### ✅ 성능 특성

1. O(n) 복잡도로, n은 최대 파일 디스크립터 번호입니다.
2. O(n) 복잡도로, n은 pollfd 배열의 크기입니다.
3. 대규모 연결에서 성능이 저하되며, 매번 전체 리스트를 확인해야 합니다.
4. 모든 파일 디스크립터를 순회하며 확인하기 때문입니다.

### ✅ 사용 패턴

1. fd_set 초기화 → 관찰할 fd 추가 → select() 호출 → 반환된 fd 처리 → 반복입니다.
2. pollfd 배열 초기화 → poll() 호출 → revents 확인 → 처리 → 반복입니다.
3. select/poll로 준비 상태를 확인하고, 논블로킹으로 실제 I/O를 수행하기 위해서입니다.

### ✅ 에러 처리 / 예외 상황

1. 잘못된 인자, 시스템 리소스 부족, 인터럽트 등에서 실패합니다.
2. 0을 반환하며, 타임아웃 시간이 지났음을 의미합니다.
3. select() 호출 후 fd_set이 수정되므로 매번 재설정해야 합니다.

### ✅ 실무 / 설계 판단

1. 소규모 연결에서는 select, 더 많은 연결이나 유연성이 필요하면 poll을 사용합니다.
2. epoll(Linux), kqueue(BSD/macOS), IOCP(Windows) 등이 있습니다.
3. 네, 소규모 연결에서는 충분히 효율적입니다.

### ✅ 요약 질문

1. select는 fd_set과 제한된 파일 디스크립터를 사용하고, poll은 배열과 무제한 파일 디스크립터를 사용합니다.
