### ✅ std::move 사용 시 주의사항

11. 이동된 객체는 미정의 상태이므로 사용하면 안 됩니다.
12. 여러 번 호출해도 문제없지만, 이미 이동된 객체는 의미가 없습니다.
13. const 객체는 이동할 수 없으므로 복사 생성자가 호출됩니다.
14. 지역 변수를 반환할 때는 RVO가 적용되므로 불필요할 수 있습니다.

### ✅ std::move와 성능

15. 불필요한 복사를 제거하여 성능을 향상시킬 수 있습니다.
16. 불필요한 복사가 발생하여 성능 저하가 발생합니다.
17. 아니요, 이동이 의미 있는 경우에만 사용해야 합니다.

### ✅ std::move와 예외 안전성

18. 네, 이동 후 원본 객체의 상태를 고려해야 합니다.
19. 부분적으로 이동된 상태가 되어 일관성이 깨질 수 있습니다.

### ✅ 실무 / 설계 판단

20. 객체를 이동할 수 있도록 rvalue로 변환해야 할 때 사용합니다.
21. 지역 변수 반환(RVO 적용), const 객체, 이미 이동된 객체에는 사용하지 않습니다.
22. 객체를 rvalue로 변환하여 이동 가능함을 표현하는 함수입니다.
