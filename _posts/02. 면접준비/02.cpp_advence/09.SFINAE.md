## 📝 모범 답변

### ✅ 기본 개념

1. 템플릿 인스턴스화 과정에서 치환 실패가 발생해도 에러가 아닌 것으로 처리하는 C++ 규칙입니다.
2. Substitution Failure Is Not An Error (치환 실패는 에러가 아니다)
3. 템플릿 인자 치환 과정에서 타입이나 표현식이 유효하지 않을 때 발생합니다.

### ✅ SFINAE 동작 원리

1. 템플릿 인자를 치환하는 과정에서 실패하면 해당 템플릿을 후보에서 제외하고 다른 후보를 시도합니다.
2. 아니요, SFINAE는 에러가 아니라 해당 템플릿을 후보에서 제외하는 것입니다.
3. SFINAE는 템플릿 후보에서만 제외하고, 일반 컴파일 에러는 전체 컴파일을 중단합니다.

### ✅ SFINAE 활용

1. SFINAE를 이용해 컴파일 타임에 타입 특성을 확인하고 조건부로 코드를 생성하는 기법입니다.
2. `std::enable_if`는 SFINAE 원리를 활용하여 조건부로 타입을 활성화/비활성화하는 도구입니다.
3. 조건이 참이면 타입을 제공하고, 거짓이면 치환 실패를 일으켜 SFINAE로 제외됩니다.
4. 타입 특성(traits)과 `std::enable_if`를 조합하여 타입이 특정 조건을 만족하는지 확인합니다.
5. `std::enable_if`를 사용하여 조건에 따라 다른 함수 오버로드를 선택합니다.

### ✅ SFINAE vs Concepts

1. SFINAE는 복잡한 템플릿 메타프로그래밍 기법이고, Concepts는 타입 제약을 명시적으로 표현하는 C++20 기능입니다.
2. 네, Concepts는 SFINAE의 많은 사용 사례를 더 명확하고 간단하게 대체할 수 있습니다.
3. C++20 이상에서는 Concepts를 우선 사용하고, 하위 호환성이 필요하거나 특수한 경우에만 SFINAE를 사용합니다.

### ✅ 실무 / 설계 판단

1. 복잡한 에러 메시지, 컴파일 시간 증가, 가독성 저하를 주의해야 합니다.
2. 복잡한 문법, 긴 컴파일 시간, 이해하기 어려운 에러 메시지가 단점입니다.
3. 템플릿 치환 실패를 에러가 아닌 후보 제외로 처리하는 C++ 규칙입니다.
