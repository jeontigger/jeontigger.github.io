## 📝 모범 답변

### ✅ 기본 개념

1. DirectX 9, DirectX 11, DirectX 12가 주요 버전입니다. 각각 2002년, 2009년, 2015년에 출시되었습니다.

2. DirectX 9: Shader Model 2.0/3.0 지원, 고수준 API. DirectX 11: 멀티스레딩 강화, Compute Shader 추가. DirectX 12: Low-level API로 CPU 오버헤드 감소.

3. 타겟 플랫폼, 하드웨어 요구사항, 개발 복잡도, 성능 요구사항, 팀의 경험 수준을 고려해야 합니다.

### ✅ DirectX 9 특징

1. 고수준 API로 사용이 쉬우며, Shader Model 2.0/3.0을 지원하고, 고정 파이프라인과 프로그래밍 가능한 셰이더를 모두 지원합니다.

2. 사용이 쉽고, 광범위한 하드웨어 호환성, 안정적인 API, 풍부한 학습 자료가 있습니다.

3. CPU 오버헤드가 크고, 멀티스레딩 지원이 약하며, 최신 하드웨어 기능을 활용하기 어렵습니다.

4. 셰이더 프로그래밍의 기능과 제약을 정의하며, 버전이 높을수록 더 많은 기능을 사용할 수 있습니다.

### ✅ DirectX 11 특징

1. Device와 Context 분리, Compute Shader 추가, 멀티스레딩 지원 강화, Tessellation 지원이 주요 변화입니다.

2. 멀티스레딩 환경에서 Context를 여러 스레드에서 사용할 수 있게 하여 성능을 향상시킵니다.

3. GPU에서 범용 계산을 수행할 수 있게 하여 물리 시뮬레이션, 이미지 처리 등에 활용됩니다.

4. Deferred Context를 사용하여 여러 스레드에서 명령을 기록하고, Immediate Context에서 실행합니다.

5. Geometry Shader, Hull/Domain Shader(Tessellation), Compute Shader가 추가되었습니다.

### ✅ DirectX 12 특징

1. CPU 오버헤드를 줄이고 개발자가 GPU를 더 세밀하게 제어할 수 있도록 하여 성능을 극대화하기 위함입니다.

2. Low-level API, Command Queue/List 분리, Resource Barrier 명시적 관리, Descriptor Heap, Multi-threaded command recording이 주요 특징입니다.

3. 장점: CPU 오버헤드 감소, 세밀한 제어, 멀티스레딩 활용 용이. 단점: 복잡도 증가, 개발 시간 증가, 에러 가능성 증가.

4. Command Queue는 GPU에 명령을 제출하는 큐이고, Command List는 기록된 명령들의 집합입니다.

5. GPU가 리소스의 상태를 추적하고 올바른 순서로 접근할 수 있도록 명시적으로 상태 전환을 관리하기 위함입니다.

### ✅ 버전별 비교

1. Device와 Context 분리, Compute Shader 추가, 멀티스레딩 지원 강화가 가장 큰 차이점입니다.

2. High-level API에서 Low-level API로 변화, CPU 오버헤드 감소, 개발 복잡도 증가가 가장 큰 차이점입니다.

3. DirectX 9: CPU 오버헤드 큼. DirectX 11: 중간 수준. DirectX 12: CPU 오버헤드 최소화, GPU 활용 극대화.

4. DirectX 9: 쉬움. DirectX 11: 중간. DirectX 12: 어려움 (Low-level 제어 필요).

5. DirectX 9: 광범위한 하드웨어 지원. DirectX 11: Windows 7 이상. DirectX 12: Windows 10 이상, 최신 하드웨어 필요.

### ✅ 버전별 사용 시점

1. 레거시 프로젝트 유지보수, 광범위한 하드웨어 지원이 필요한 경우, 빠른 프로토타이핑이 필요한 경우에 사용합니다.

2. 현대적인 기능이 필요하지만 복잡도를 낮추고 싶을 때, 멀티스레딩 지원이 필요할 때 사용합니다.

3. 최대 성능이 필요하고, CPU 오버헤드를 최소화해야 하며, 개발 복잡도를 감수할 수 있을 때 사용합니다.

4. 성능 병목이 발생하거나, 최신 하드웨어 기능이 필요하거나, 유지보수 비용이 증가할 때 고려합니다.

### ✅ 성능 / 최적화

1. DirectX 9는 CPU 오버헤드가 크고, DirectX 11은 중간 수준이며, DirectX 12는 CPU 오버헤드가 최소화되어 GPU 활용도가 높습니다.

2. Multi-threaded command recording, Resource Barrier 최적화, Descriptor Heap 효율적 사용, GPU 타임라인 최적화가 필요합니다.

3. DirectX 9: Draw Call당 오버헤드 큼. DirectX 11: 중간 수준. DirectX 12: Draw Call 오버헤드 최소화 가능.

### ✅ 실무 / 설계 판단

1. 타겟 플랫폼, 하드웨어 요구사항, 개발 일정, 팀 경험, 성능 요구사항을 종합적으로 고려해야 합니다.

2. API 변경사항 학습, 리소스 관리 방식 변경, 멀티스레딩 구조 재설계, 테스트 범위 확대가 필요합니다.

3. 장점: 성능 향상, 최신 하드웨어 활용. 단점: 개발 복잡도 증가, 학습 곡선, 버그 가능성 증가.

### ✅ 요약 질문

1. DirectX 9는 고수준 API로 사용이 쉽지만 CPU 오버헤드가 큽니다. DirectX 11은 Device/Context 분리와 Compute Shader로 멀티스레딩과 성능이 향상되었습니다. DirectX 12는 Low-level API로 CPU 오버헤드를 최소화하지만 개발 복잡도가 높습니다. 프로젝트 요구사항과 팀 역량에 따라 선택해야 합니다.
