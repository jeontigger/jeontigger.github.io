# 🧠 Instancing 개념 설명 가능 면접 질문 리스트

## ✅ 기본 개념

1. Instancing이란 무엇인가요?

    - 인스턴싱이란 같은 매쉬, 같은 머터리얼을 사용하는 객체를 여러개 그려야 할 때 한 번의 드로우콜로 해결하는 최적화 기법입니다.

2. Instancing은 어떤 성능 문제를 해결하기 위한 기법인가요?

    - 드로우콜 병목현상을 해결하기 위함입니다.

## ✅ Draw Call과의 관계 (핵심)

1. Instancing은 Draw Call 수에 어떤 영향을 주나요?

    - 인스턴싱은 드로우콜 수가 줄어듭니다. 

2. Instancing이 CPU 성능에 유리한 이유는 무엇인가요?

    - 드로우 콜 수를 줄이기 때문에 CPU 작업이 줄어듭니다.

## ✅ 동작 방식

1. Instancing은 내부적으로 무엇을 "반복"하나요?

    - 드로우를 반복합니다.

2. 어떤 조건을 만족해야 Instancing이 가능한가요?

    - 같은 매쉬, 같은 머터리얼, 같은 파이프라인 상태를 만족해야 합니다.

## ✅ Batching과의 비교 (핵심)

1. Instancing과 Batching의 가장 큰 차이는 무엇인가요?

    - 인스턴싱은 같은 매쉬끼리, 배칭은 다른 매쉬를 하나로 만드는 작업입니다.

2. Instancing이 Batching보다 유리한 경우는 언제인가요?

    - 같은 물체를 여러번 그려야할 때 유리합니다.

## ✅ 인스턴스 데이터

1. Instancing에서 전달하는 인스턴스 데이터에는 무엇이 있나요?

    - 객체별 위치, 상태 등을 전달합니다.

2. 인스턴스별로 다른 정보를 전달하려면 어떻게 해야 하나요?

    - 다른 묶음으로 묶입니다.

## ✅ 메모리 / GPU 관점

1. Instancing의 메모리 사용은 어떻게 되나요?

    - 메모리 기준으론 변화가 없습니다.

2. Instancing이 GPU에서 효율적인 이유는 무엇인가요?

    - 계산을 한꺼번에 진행할 수 있기 때문입니다.

## ✅ 제약 / 한계

1. 서로 다른 메쉬를 Instancing으로 그릴 수 있나요?

    - 없습니다.

2. Instancing이 항상 성능을 향상시키지 않는 이유는 무엇인가요?

    - 상태가 달라지면 인스턴싱이 불가능하기 때문입니다.
