# 🧠 좌표계(Local / World / View / Projection) 설명 가능 면접 질문 리스트

## ✅ 기본 개념

1. 좌표계란 무엇인가요?

    - 좌표계란 정점을 바라보는 기준입니다.

2. 렌더링에서 좌표계를 여러 단계로 나누는 이유는 무엇인가요?

    - 좌표계의 변환에 따라 상태가 달라지고, 나눈 단계마다 할 수 있는 역할이 나뉘어있기 때문입니다.

## ✅ Local Space

1. Local Space란 무엇인가요?

    - 모델 좌표계라고도 합니다. 모델의 원점을 기준으로 정점들의 상대적 위치를 나타낸 좌표계입니ㅏㄷ.

2. Local Space 좌표는 어떤 기준을 가지나요?

    - 모델의 원점입니다.

## ✅ World Space

1. World Space란 무엇인가요?

    - 모델을 게임 엔진 상의 월드 위에 올려놓은 좌표계입니다.

2. Local → World 변환은 어떤 정보를 반영하나요?

    - 해당 모델의 정점들이 월드 위치로 변환됩니다.

## ✅ View Space (Camera Space)

1. View Space란 무엇인가요?

    - 카메라 기준의 좌표계입니다.

2. World → View 변환이 의미하는 것은 무엇인가요?

    - 월드상 놓여있는 정점들의 좌표를 view 변환을 통해 카메라 기준으로 상대적 좌표로 변경됩니다.

## ✅ Projection Space

1. Projection Space란 무엇인가요?

    - 카메라에 사영된 결과 좌표계입니다. 

2. Perspective Projection과 Orthographic Projection의 차이는 무엇인가요?

    - Perspective는 원근감이 존재하는 사영법이고, Orthographic은 원근감이 없어서 주로 2D에서 사용하는 사영법입니다.

## ✅ 변환 행렬 / 순서 (핵심)

1. 좌표 변환은 어떤 순서로 이루어지나요?

    - local->world->view->projection 순서로 이루어집니다.

2. 왜 이 순서를 지켜야 하나요?

    - 논리적 흐름이 모델->월드->카메라->투영 순으로 이뤄지기 때문입니다.

## ✅ 실무 / 셰이더 관점

1. MVP 행렬이란 무엇인가요?

    - 모델 뷰 프로젝션 변환 행렬이 모두 합쳐져 있는 행렬입니다.

2. Vertex Shader에서 좌표계 변환은 보통 어디까지 수행하나요?

    - 보통 프로젝션 변환까지 수행합니다.

### 📊 전체 평가

**강점:**
- ✅ 좌표계 변환 순서를 정확히 알고 있음 (Local → World → View → Projection)
- ✅ MVP 행렬의 개념을 이해함
- ✅ Vertex Shader에서 프로젝션 변환까지 수행한다는 것을 이해함
- ✅ Perspective와 Orthographic의 기본적인 차이를 이해함

**개선 필요:**
- ⚠️ **부정확**: 좌표계 정의가 부정확 ("정점을 바라보는 기준")
- ⚠️ **부정확**: Local Space 좌표 기준 설명 부족 (방향 누락)
- ⚠️ **부정확**: Local → World 변환 정보 설명 부족 (위치, 회전, 스케일)
- ⚠️ **부정확**: Orthographic Projection 설명 부정확 ("주로 2D에서 사용")
- ⚠️ **부정확**: 좌표 변환 순서를 지켜야 하는 이유 설명 부족
- ⚠️ **오타**: "입니ㅏㄷ" → "입니다"
- ❌ **답변 누락**: 요약 질문

**우선순위:**
1. **높음**: 
   - 좌표계 정의 정확화
   - Local → World 변환 정보 정확화 (위치, 회전, 스케일)
   - 좌표 변환 순서를 지켜야 하는 이유 정확화
2. **중간**: 
   - Local Space 좌표 기준 보완 (방향)
   - Orthographic Projection 설명 정정
   - 요약 질문 답변
3. **낮음**: 
   - 오타 수정

**특별 주의사항:**
- **좌표계 정의**: 좌표계는 점의 위치를 표현하는 기준 공간입니다. 정점뿐만 아니라 모든 점에 적용됩니다.
- **Local → World 변환**: 위치(translation), 회전(rotation), 스케일(scale) 정보를 반영합니다. 이는 면접에서 자주 나오는 질문입니다.
- **변환 순서**: 각 좌표계가 이전 좌표계를 기준으로 정의되므로 순서를 바꿀 수 없습니다. 이는 수학적으로 필수적입니다.
- **Orthographic Projection**: 원근감이 없는 투영 방식이며, 2D뿐만 아니라 3D에서도 사용됩니다(UI, 디버그 뷰 등).
- **MVP 행렬**: Model, View, Projection 행렬의 곱이며, 행렬 곱셈 순서는 오른쪽에서 왼쪽입니다 (MVP = P × V × M).
