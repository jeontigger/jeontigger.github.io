## 📝 모범 답변

### ✅ 기본 개념

1. 미리 할당한 메모리 블록을 재사용하는 메모리 관리 기법입니다.
2. 할당/해제 비용 감소, 단편화 감소, 예측 가능한 성능을 위해서입니다.

### ✅ 동작 방식 (핵심)

3. 미리 큰 메모리 블록을 할당하고 필요할 때 나눠주는 방식입니다.
4. 풀에서 사용 가능한 블록을 찾아 반환합니다.
5. 블록을 풀로 반환해 재사용 가능하게 만듭니다.

### ✅ 성능 이점

6. 시스템 호출 감소, 할당 알고리즘 단순화, 캐시 효율 향상 때문입니다.
7. 고정 크기 블록 사용과 재사용으로 단편화가 줄어듭니다.

### ✅ 단점 / 제약

8. 초기 메모리 사용량 증가, 유연성 감소, 구현 복잡도 증가입니다.
9. 할당 크기가 다양하거나 메모리 사용 패턴이 불규칙할 때입니다.

### ✅ 구현 방식

10. 모든 블록이 동일한 크기인 메모리 풀입니다.
11. 다양한 크기의 블록을 관리하는 메모리 풀입니다.
12. 고정 크기는 빠르고 단순하지만 낭비가 있고, 가변 크기는 유연하지만 복잡합니다.

### ✅ 실무 사용

13. 게임 엔진, 임베디드 시스템, 고성능 서버입니다.
14. 프레임 단위 할당/해제와 성능 예측이 중요하기 때문입니다.

### ✅ placement new 연계

15. 메모리 풀에서 할당한 메모리에 객체를 생성할 때 사용합니다.
