## 📝 모범 답변

### ✅ 기본 개념

1. 메모리 누수를 자동으로 찾아주는 디버깅 도구입니다.
2. 수동으로 찾기 어렵고 시간이 많이 걸리기 때문입니다.

### ✅ Valgrind

3. 동적 분석 도구로 메모리 오류를 탐지합니다.
4. 메모리 누수, 사용 후 해제, 초기화되지 않은 메모리 접근을 탐지합니다.
5. 실행 속도가 크게 느려지고(10-50배) 가상 머신에서 동작합니다.

### ✅ AddressSanitizer (ASan)

6. 컴파일 타임에 코드를 계측해 런타임에 메모리 오류를 탐지하는 도구입니다.
7. 컴파일 타임 계측이라 Valgrind보다 훨씬 빠릅니다(2-3배).
8. 약 2배의 메모리 오버헤드가 발생합니다.

### ✅ 메모리 프로파일러

9. Visual Studio Diagnostic Tools, Xcode Instruments, Dr. Memory 등이 있습니다.
10. 메모리 할당 위치, 크기, 누수 지점, 호출 스택 정보입니다.

### ✅ 실무 디버깅 방법

11. 도구로 누수 지점을 찾고, 호출 스택을 분석해 원인을 파악합니다.
12. 재현 → 도구 실행 → 누수 지점 확인 → 코드 분석 → 수정 → 재검증입니다.

### ✅ 도구 비교

13. Valgrind는 느리지만 정확하고, ASan은 빠르지만 컴파일러 의존적입니다.
14. 개발 중에는 ASan, 정밀 분석에는 Valgrind를 사용합니다.

### ✅ 성능 / 오버헤드

15. Valgrind는 10-50배, ASan은 2-3배 느려집니다.
16. ASan은 가능하지만 오버헤드가 있어 주의가 필요합니다.

### ✅ 요약 질문

17. 메모리 누수 탐지 도구는 "메모리 누수를 자동으로 찾아주는 디버깅 도구"입니다.
