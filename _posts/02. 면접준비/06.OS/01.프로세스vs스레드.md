# 🧠 프로세스(Process) vs 스레드(Thread) 차이 설명 가능 면접 질문 리스트

## ✅ 기본 개념

1. 프로세스란 무엇인가요?

    - 프로세스란 프로그램이 실행되고 있는 상태입니다. 하나의 프로그램은 하나 이상의 프로세스가 필요합니다. 프로세스는 실행되면서 운영체제에게 독립적인 가상메모리를 부여받습니다. 

2. 스레드란 무엇인가요?

    - 스레드란 프로세스 내에서 실행되는 가장 작은 실행 단위입니다. 하나의 프로세스는 하나 이상의 스레드로 구성되어있습니다. 스레드는 프로세스가 부여받은 가상 메모리의 코드, 데이터, 힙 영역을 공유하고 스택 메모리만 별도로 할당받게 됩니다. 따라서 스레드간의 정보 공유는 상대적으로 빠르게 수행 가능합니다. 그러나 같은 메모리 공간을 사용하기 때문에 동기화에 대한 문제를 해결해야합니다.

3. 프로세스와 스레드의 가장 큰 차이는 무엇인가요?

    - 가용 메모리를 얼마나 부여받는지, 접근성과 안정성에 차이가 큽니다. 

## ✅ 메모리 구조

1. 프로세스는 어떤 메모리 공간을 가지나요?

    - 프로세스는 가상메모리 공간 전체를 부여받습니다. 코드, 데이터, 힙, 스택 영역을 가집니다.

2. 스레드는 어떤 메모리 자원을 공유하나요?

    - 스레드는 가상 메모리 공간의 코드, 데이터, 힙 영역을 공유하고 독립적인 스택 영역을 가집니다.

3. 스레드마다 독립적인 자원은 무엇인가요?

    - 스택 영역입니다. 스레드마다 고유한 스택 영역을 가집니다.

## ✅ 생성 / 전환 비용

1. 프로세스 생성 비용이 큰 이유는 무엇인가요?

    - 프로세스는 가상 메모리 공간을 모두 부여받기 때문에 생성 비용이 큽니다. 

2. 스레드 생성이 프로세스보다 가벼운 이유는 무엇인가요?

    - 스레드는 프로세스에 비해서 스택 영역만 독립적이기 때문에 PCB보다 가벼운 TCB만 생성하면 되기 때문에 가볍습니다.

3. 컨텍스트 스위칭 비용은 왜 차이가 나나요?

    - 컨텍스트 스위칭을 할 때 프로세스끼리는 PCB를 교체해야하고, 스레드끼리는 TCB를 교체해야 합니다. 이 때 TCB가 PCB보다 요구되는 정보가 적기 때문에 더 가벼운 TCB를 교체할 때 스위칭 비용이 저렴해집니다.

## ✅ 통신 방식

1. 프로세스 간 통신(IPC)은 왜 복잡한가요?

    - 프로세스끼리는 독립된 메모리 공간을 가지기 때문에 메모리를 공유하기 위해 운영체제의 개입이 필요하게 됩니다. 운영체제의 개입으로 성능 또한 상대적으로 느립니다.

2. 스레드 간 통신은 왜 상대적으로 쉬운가요?

    - 스레드간 메모리는 코드, 데이터, 힙 영역을 공유하기 때문에 같은 메모리 영역을 접근하면 되므로 IPC보다는 상대적으로 쉽다고 볼 수 있습니다.

## ✅ 안정성 / 오류 전파

1. 하나의 프로세스가 죽으면 다른 프로세스에 영향이 있나요?

    - 각각의 프로세스는 독립적이기 때문에 다른 프로세스에게 영향을 미치지 않습니다.

2. 하나의 스레드 오류는 어떤 영향을 미치나요?

    - 하나의 스레드가 오류가 발생한다면, 같은 메모리 공간을 사용하는 다른 스레드나, 스레드를 운용하는 프로세스 자체에 영향을 미칠 수 있습니다.

## ✅ 동기화 / 경쟁 조건

1. 스레드에서 동기화가 필요한 이유는 무엇인가요?

    - 같은 메모리 공간을 사용하기 때문에 메모리 정합성이 필요합니다.

2. 프로세스에서도 동기화 문제가 발생하나요?

    - 프로세스간에는 동기화 문제가 발생하지 않습니다.

## ✅ 사용 판단

1. 어떤 경우에 프로세스를 사용하는 것이 적절한가요?

    - 메모리 자원을 넉넉하게 사용하여 더 안정적이고 독립적인 시스템을 구축해야할 때 프로세스를 사용하고 멀티프로세스 프로그램을 만드는 것이 적절합니다.

2. 어떤 경우에 스레드를 사용하는 것이 적절한가요?

    - 컨텍스트 스위칭이 자주 일어나며, 하는 작업이 단순하고 동기화 해야하는 영역이 충분히 작을 때 사용하는 것이 적절합니다.

## ✅ 요약 질문

1. 프로세스와 스레드의 차이를 한 문장으로 요약해보세요.

    - 프로세스는 모든 메모리 영역을 부여받고, 프로세스간 독립적인 공간을 사용합니다. 따라서 많은 자원이 필요하고, 안정적입니다. 스레드는 프로세스 내의 단위로써 가상 메모리 영역의 코드, 힙, 데이터 영역은 공유하고 스택 영역만 별도로 할당받습니다. 따라서 메모리 공유는 가볍고 빠르게 진행되지만 동기화에 대한 작업이 필요합니다.

  - 💡 **보완**: 프로세스 간 컨텍스트 스위칭은 주소 공간(페이지 테이블)을 전환해야 하므로 TLB 플러시가 발생하고 비용이 큽니다. 스레드 간 컨텍스트 스위칭은 같은 주소 공간을 사용하므로 주소 공간 전환이 필요 없어 비용이 적습니다.

### ✅ 통신 방식

**질문 1: 프로세스 간 통신(IPC)은 왜 복잡한가요?**
- **현재 답변**: "프로세스끼리는 독립된 메모리 공간을 가지기 때문에 메모리를 공유하기 위해 운영체제의 개입이 필요하게 됩니다. 운영체제의 개입으로 성능 또한 상대적으로 느립니다."
- **피드백**:
  - ⚠️ **부정확**: "메모리를 공유하기 위해"보다는 "주소 공간이 분리되어 있어서"가 더 정확합니다.
  - ✅ **정확한 답변**: "주소 공간이 분리되어 있기 때문입니다."
  - 💡 **보완**: 프로세스는 독립적인 주소 공간을 가지므로 직접 메모리 접근이 불가능합니다. 공유 메모리, 파이프, 소켓 등 운영체제가 제공하는 IPC 메커니즘을 사용해야 하므로 복잡하고 비용이 큽니다.

**질문 2: 스레드 간 통신은 왜 상대적으로 쉬운가요?**
- **현재 답변**: "스레드간 메모리는 코드, 데이터, 힙 영역을 공유하기 때문에 같은 메모리 영역을 접근하면 되므로 IPC보다는 상대적으로 쉽다고 볼 수 있습니다."
- **피드백**:
  - 💡 **보완**: 정확합니다. "같은 주소 공간을 사용하기 때문입니다"라고 간결하게 표현할 수 있습니다.

### ✅ 안정성 / 오류 전파

**질문 1: 하나의 프로세스가 죽으면 다른 프로세스에 영향이 있나요?**
- **현재 답변**: "각각의 프로세스는 독립적이기 때문에 다른 프로세스에게 영향을 미치지 않습니다."
- **피드백**:
  - ⚠️ **부정확**: 일반적으로 맞지만, 부모-자식 프로세스 관계나 시그널을 통한 통신 등 예외적인 경우도 있습니다.
  - ✅ **정확한 답변**: "보통 영향을 주지 않습니다."
  - 💡 **보완**: 프로세스는 독립적인 주소 공간을 가지므로 일반적으로 한 프로세스의 종료가 다른 프로세스에 직접 영향을 주지 않습니다. 하지만 부모-자식 관계나 IPC를 통한 간접적인 영향은 있을 수 있습니다.

**질문 2: 하나의 스레드 오류는 어떤 영향을 미치나요?**
- **현재 답변**: "하나의 스레드가 오류가 발생한다면, 같은 메모리 공간을 사용하는 다른 스레드나, 스레드를 운용하는 프로세스 자체에 영향을 미칠 수 있습니다."
- **피드백**:
  - 💡 **보완**: 정확합니다. "프로세스 전체가 비정상 종료될 수 있습니다"라고 표현하면 더 명확합니다. 한 스레드의 치명적 오류(세그먼트 오류 등)는 프로세스 전체를 종료시킬 수 있습니다.

### ✅ 동기화 / 경쟁 조건

**질문 1: 스레드에서 동기화가 필요한 이유는 무엇인가요?**
- **현재 답변**: "같은 메모리 공간을 사용하기 때문에 메모리 정합성이 필요합니다."
- **피드백**:
  - ⚠️ **부정확**: "메모리 정합성"보다는 "공유 자원에 대한 경쟁 조건"이 더 정확합니다.
  - ✅ **정확한 답변**: "공유 자원에 대한 경쟁 조건이 발생하기 때문입니다."
  - 💡 **보완**: 여러 스레드가 같은 자원(변수, 자료구조 등)에 동시에 접근할 때 경쟁 조건(race condition)이 발생할 수 있습니다. 이를 방지하기 위해 뮤텍스, 세마포어 등의 동기화 메커니즘이 필요합니다.

**질문 2: 프로세스에서도 동기화 문제가 발생하나요?**
- **현재 답변**: "프로세스간에는 동기화 문제가 발생하지 않습니다."
- **피드백**:
  - ❌ **오개념**: 프로세스 간에는 주소 공간이 분리되어 있어 직접적인 동기화 문제는 없지만, IPC를 통해 공유하는 자원에 대해서는 동기화가 필요합니다.
  - ✅ **정확한 답변**: "네, IPC 자원에 대해 발생할 수 있습니다."
  - 💡 **보완**: 프로세스 간에는 메모리를 직접 공유하지 않으므로 같은 메모리 접근으로 인한 경쟁 조건은 없습니다. 하지만 공유 메모리, 파일, 소켓 등 IPC를 통해 공유하는 자원에 대해서는 동기화가 필요합니다.

### ✅ 사용 판단

**질문 1: 어떤 경우에 프로세스를 사용하는 것이 적절한가요?**
- **현재 답변**: "메모리 자원을 넉넉하게 사용하여 더 안정적이고 독립적인 시스템을 구축해야할 때 프로세스를 사용하고 멀티프로세스 프로그램을 만드는 것이 적절합니다."
- **피드백**:
  - ⚠️ **부정확**: "메모리 자원을 넉넉하게 사용"이라는 표현이 모호합니다. 핵심은 격리와 안정성입니다.
  - ✅ **정확한 답변**: "강한 격리와 안정성이 필요할 때입니다."
  - 💡 **보완**: 프로세스는 다음 경우에 적절합니다:
    - 강한 격리와 안정성이 필요한 경우(한 프로세스의 오류가 다른 프로세스에 영향을 주지 않음)
    - 독립적인 실행 환경이 필요한 경우
    - 보안이 중요한 경우

**질문 2: 어떤 경우에 스레드를 사용하는 것이 적절한가요?**
- **현재 답변**: "컨텍스트 스위칭이 자주 일어나며, 하는 작업이 단순하고 동기화 해야하는 영역이 충분히 작을 때 사용하는 것이 적절합니다."
- **피드백**:
  - ⚠️ **부정확**: "컨텍스트 스위칭이 자주 일어나며"는 스레드를 사용하는 이유가 아니라 결과입니다. 병렬 처리와 자원 공유가 핵심입니다.
  - ✅ **정확한 답변**: "가벼운 병렬 처리와 자원 공유가 필요할 때입니다."
  - 💡 **보완**: 스레드는 다음 경우에 적절합니다:
    - 가벼운 병렬 처리가 필요한 경우(생성 비용이 적음)
    - 자원 공유와 빠른 통신이 필요한 경우
    - I/O 바운드 작업의 동시 처리

### ✅ 요약 질문

**질문 1: 프로세스와 스레드의 차이를 한 문장으로 요약해보세요.**
- **현재 답변**: "프로세스는 모든 메모리 영역을 부여받고, 프로세스간 독립적인 공간을 사용합니다. 따라서 많은 자원이 필요하고, 안정적입니다. 스레드는 프로세스 내의 단위로써 가상 메모리 영역의 코드, 힙, 데이터 영역은 공유하고 스택 영역만 별도로 할당받습니다. 따라서 메모리 공유는 가볍고 빠르게 진행되지만 동기화에 대한 작업이 필요합니다."
- **피드백**:
  - ⚠️ **부정확**: "한 문장으로 요약"이라는 요구사항을 충족하지 못했습니다. 여러 문장으로 나뉘어 있습니다.
  - ✅ **정확한 답변**: "프로세스는 '격리된 실행 단위', 스레드는 '공유된 실행 흐름'입니다."
  - 💡 **보완**: 프로세스는 독립적인 주소 공간을 가진 격리된 실행 단위이고, 스레드는 같은 주소 공간을 공유하는 실행 흐름입니다. 이에 따라 자원 사용, 통신 비용, 안정성이 달라집니다.

### 📊 전체 평가

**강점:**
- 프로세스와 스레드의 기본 개념을 이해하고 있습니다.
- 메모리 구조 차이를 이해하고 있습니다.
- 통신 방식과 안정성 차이를 알고 있습니다.

**개선 필요:**
- ❌ 프로세스 동기화 문제 설명 오개념 (IPC 자원 동기화 필요성 누락)
- ⚠️ 프로세스 정의 설명 부정확 ("실행 상태" vs "독립적 인스턴스")
- ⚠️ 스레드 독립적 자원 설명 부족 (레지스터 상태 누락)
- ⚠️ 프로세스 생성 비용 설명 부정확 (주소 공간과 자원 초기화)
- ⚠️ 스레드 생성 비용 설명 부정확 (주소 공간 공유)
- ⚠️ 컨텍스트 스위칭 비용 설명 부족 (주소 공간 전환)
- ⚠️ 프로세스 간 통신 복잡성 설명 부정확 ("메모리 공유" vs "주소 공간 분리")
- ⚠️ 프로세스 오류 전파 설명 부정확 (예외적 경우 누락)
- ⚠️ 스레드 동기화 필요성 설명 부정확 ("메모리 정합성" vs "경쟁 조건")
- ⚠️ 프로세스 사용 판단 기준 설명 모호
- ⚠️ 스레드 사용 판단 기준 설명 부정확 ("컨텍스트 스위칭이 자주" vs "병렬 처리")
- ⚠️ 요약 질문이 한 문장이 아님

**우선순위:**
1. **높음**: 프로세스 동기화 문제 오개념 수정 (IPC 자원 동기화 필요)
2. **높음**: 컨텍스트 스위칭 비용 설명 수정 (주소 공간 전환)
3. **높음**: 프로세스/스레드 정의 명확화
4. **중간**: 생성 비용 설명 수정 (주소 공간 초기화 vs 공유)
5. **중간**: 통신 방식 설명 수정
6. **중간**: 사용 판단 기준 명확화
7. **낮음**: 표현 정확성 개선

**특별 주의사항:**
- 프로세스는 "격리된 실행 단위", 스레드는 "공유된 실행 흐름"입니다. 핵심 차이는 자원 공유 범위와 독립성입니다.
- 프로세스 간 컨텍스트 스위칭은 주소 공간 전환이 필요하므로 TLB 플러시가 발생하고 비용이 큽니다. 스레드 간 컨텍스트 스위칭은 같은 주소 공간을 사용하므로 비용이 적습니다.
- 프로세스 간에는 메모리를 직접 공유하지 않지만, IPC를 통해 공유하는 자원(공유 메모리, 파일 등)에 대해서는 동기화가 필요합니다.
- 스레드는 스택과 레지스터 상태를 독립적으로 가지며, 코드/힙/데이터 영역을 공유합니다.
