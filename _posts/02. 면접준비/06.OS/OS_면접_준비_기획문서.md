### 1.2 파일별 상세 분석

#### `01.프로세스vs스레드.md` - ✅ 양호

**강점:**
- GPT 답변이 완성되어 있음
- 기본 개념 → 메모리 구조 → 생성/전환 비용 → 통신 방식 → 안정성/오류 전파 → 동기화/경쟁 조건 → 사용 판단 → 요약 순서로 잘 구성됨
- 프로세스와 스레드의 차이점이 명확함

**개선 필요 사항:**
1. **사용자 답변 공간 없음**
   - 일관성을 위해 추가 고려 (선택사항)

#### `02.컨텍스트스위칭.md` - ✅ 양호

**강점:**
- GPT 답변이 완성되어 있음
- 기본 개념 → 저장/복원 대상 (핵심) → 프로세스 vs 스레드 → OS/스케줄링 연계 → 성능 관점 → 동기화/블로킹 → 설계 판단 → 요약 순서로 잘 구성됨
- 컨텍스트 스위칭의 비용과 원인이 명확함

**개선 필요 사항:**
1. **사용자 답변 공간 없음**
   - 일관성을 위해 추가 고려 (선택사항)

#### `03.pcb_tcb.md` - ⚠️ GPT 답변 완전히 누락

**강점:**
- 질문 구조가 잘 되어 있음: 기본 개념 → 포함 정보 (핵심) → 프로세스/스레드 관계 → 컨텍스트 스위칭 연계 → 메모리/자원 관리 → 설계 판단 → 요약
- 카테고리별로 질문이 체계적으로 구성됨

**문제점:**
1. **GPT 답변 섹션이 비어있음**
   - GPT 답변 섹션에 질문만 반복되어 있고 실제 답변이 없음
   - 모든 질문에 대한 답변 필요

2. **사용자 답변 공간 없음**
   - 다른 파일과 달리 사용자 답변 공간이 제공되지 않음
   - 일관성을 위해 추가 고려

#### `04.mutex_semaphore.md` - ✅ 양호

**강점:**
- GPT 답변이 완성되어 있음
- 기본 개념 → 소유권/잠금 규칙 → 카운트 개념 → 동작 방식 → 사용 목적 → 오류/위험 요소 → 설계 판단 → 요약 순서로 잘 구성됨
- mutex와 semaphore의 차이점이 명확함

**개선 필요 사항:**
1. **사용자 답변 공간 없음**
   - 일관성을 위해 추가 고려 (선택사항)

#### `05.deadlock.md` - ✅ 양호

**강점:**
- GPT 답변이 완성되어 있음
- 기본 개념 → Deadlock 4조건 (필수 암기) → 조건 간 관계 → 예방/회피 관점 → 요약 순서로 잘 구성됨
- Deadlock 4조건이 명확히 드러남

**개선 필요 사항:**
1. **사용자 답변 공간 없음**
   - 일관성을 위해 추가 고려 (선택사항)

2. **Deadlock 회피 알고리즘 언급 부족**
   - Banker's Algorithm 등 언급 고려

#### `06.race_condition.md` - ✅ 양호

**강점:**
- GPT 답변이 완성되어 있음
- 기본 개념 → 대표적인 발생 사례 (핵심) → 메모리/연산 단위 관점 → 체크-후-사용(TOCTOU) → 실무 사례 → 예방/해결 → 요약 순서로 잘 구성됨
- Race Condition의 다양한 사례가 잘 다뤄짐

**개선 필요 사항:**
1. **사용자 답변 공간 없음**
   - 일관성을 위해 추가 고려 (선택사항)

#### `07.atomic.md` - ✅ 양호

**강점:**
- GPT 답변이 완성되어 있음
- 기본 개념 → 원자성 보장 → atomic vs mutex → 메모리 모델 관점 → 성능/하드웨어 연계 → 사용 제한/주의점 → 요약 순서로 잘 구성됨
- atomic의 특성과 사용법이 명확함

**개선 필요 사항:**
1. **사용자 답변 공간 없음**
   - 일관성을 위해 추가 고려 (선택사항)

## 4. 질문 형식 표준화 (standardize_format)

### 4.1 권장 파일 구조

```markdown
# 🧠 [주제] 면접 질문 리스트

## ✅ [카테고리 1]

1. [질문]
   
   - [사용자 답변 공간] (선택사항)

2. [질문]
   
   - [사용자 답변 공간] (선택사항)

## ✅ [카테고리 2]

...

<details>
<summary>GPT 1줄 답변</summary>
<div markdown="1">

## 📝 모범 답변

### [카테고리 1]

1. [답변]
2. [답변]

### [카테고리 2]

...

</div>
</details>

---

## GPT 피드백

[필요시 피드백 추가]
```

### 4.2 질문 작성 가이드라인

#### 질문 수준
- **기본 개념**: 면접 초반에 나올 수 있는 기본적인 질문
- **심화 개념**: 내부 구조, 동작 원리, 알고리즘 등
- **실무 적용**: 언제 사용하는지, 어떤 경우에 선택하는지

#### 질문 순서
1. 기본 개념 질문 (2-3개)
2. 내부 구조/동작 원리 질문 (3-5개)
3. 다른 방식과의 비교 질문 (2-4개)
4. 실무 적용/선택 기준 질문 (1-2개)
5. 요약 질문 (1개)

#### 필수 포함 질문 유형
- [ ] 기본 개념 질문 포함
- [ ] 내부 구조/동작 원리 질문 포함
- [ ] 다른 방식과의 비교 질문 포함 (해당되는 경우)
- [ ] 실무 적용/선택 기준 질문 포함
- [ ] 요약 질문 포함

### 4.3 답변 작성 가이드라인

#### GPT 답변 형식
- **1줄 답변**: 간결하고 핵심만 담은 답변
- **카테고리별 구분**: 질문 카테고리와 동일하게 구성
- **번호 매칭**: 질문 번호와 답변 번호 일치

#### 답변 내용 기준
- **정확성**: 기술적으로 정확한 정보 제공
- **간결성**: 면접에서 답변할 수 있을 정도로 간결
- **완전성**: 질문의 핵심을 모두 다룸
- **예시 포함**: 필요시 코드 예시나 구체적 예시 포함

#### 사용자 답변 공간
- **선택사항**: 모든 질문에 필수는 아님
- **일관성**: 파일 내에서 일관되게 적용
- **형식**: 4칸 들여쓰기로 답변 공간 제공

### 4.4 카테고리 명명 규칙

**표준 카테고리:**
- `## ✅ 기본 개념`
- `## ✅ [구조/동작] (핵심)` (PCB/TCB, Deadlock 4조건 등)
- `## ✅ [다른 방식]과의 비교` (해당되는 경우)
- `## ✅ 실무 / 설계 판단`
- `## ✅ 요약 질문`

**주제별 추가 카테고리:**
- 프로세스 관련: `## ✅ 메모리 구조`, `## ✅ 생성 / 전환 비용`
- 동기화 관련: `## ✅ 소유권 / 잠금 규칙`, `## ✅ 동작 방식`

## 7. 참고 사항

### 파일명 규칙
- 형식: `##.주제명.md` (##는 2자리 숫자)
- 예시: `01.프로세스vs스레드.md`, `08.스케줄링알고리즘.md`

### 마크다운 형식
- 제목: `# 🧠 [주제] 면접 질문 리스트`
- 카테고리: `## ✅ [카테고리명]`
- 질문: 번호와 함께 작성
- 사용자 답변: 4칸 들여쓰기
- GPT 답변: `<details>` 태그 내부에 작성

### 질문 작성 시 주의사항
- 면접에서 실제로 나올 수 있는 수준으로 작성
- 기본 개념부터 심화 개념까지 단계적으로 구성
- OS 관련 질문은 성능과 안정성 모두 고려
- 동기화 질문은 Deadlock과 Race Condition 방지 관점 포함
