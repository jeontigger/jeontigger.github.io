# 🧠 C++ std::map vs std::unordered_map 차이 면접 질문 리스트

## ✅ 기본 개념

1. std::map과 std::unordered_map의 가장 큰 차이는 무엇인가요?

    - map은 내부적으로 이진 균형 트리로 이루어져있고, unordered_map은 해쉬 함수와 해쉬 테이블로 이루어져있습니다. 그래서 map은 내부적으로 정렬되어있고 삽입, 삭제, 접근이 O(logN)으로 이루어집니다. unordered_map은 해쉬 함수에 따라 다르지만 삽입, 삭제, 접근이 O(1)로 이루어지며 해쉬 충돌이 많이 일어나면 같은 버킷에서 다시 찾는 시간때문에 O(N)까지 증가하며 해쉬 테이블을 유지하기 위한 추가적인 메모리가 필요합니다.

## ✅ 정렬 / 순서

1. std::map은 어떤 기준으로 정렬되나요?

    - key를 기준으로 오버로딩 되어있는 비교 연산자를 기준으로 정렬됩니다.

2. std::unordered_map은 원소의 순서를 보장하나요?

    - 보장하지 않습니다. 해쉬함수로 해쉬값으로 관리하기 때문에 원소의 순서를 보장할 수 없습니다.

## ✅ 시간 복잡도

1. std::map의 평균 시간 복잡도는 어떻게 되나요?

    - 삽입 삭제 접근이 O(logN)입니다.

2. std::unordered_map의 평균 시간 복잡도는 어떻게 되나요?

    - 삽입 삭제 접근이 보통 O(1)이지만 최악의 경우 O(N)입니다.

3. 최악의 경우 시간 복잡도는 각각 어떻게 되나요?

    - map은 O(logN)이고 unordered_map은 O(N)입니다.

## ✅ 해시 / 비교 연산

1. std::unordered_map에서 key는 어떻게 비교되나요?

    - key가 해쉬함수를 거친 해쉬값으로 비교됩니다.

2. std::map에서 key 비교 방식은 무엇인가요?

    - 오버로딩된 비교 연산자로 비교됩니다.

## ✅ 메모리 / 성능 특성

1. 메모리 사용량 측면에서 두 컨테이너의 차이는 무엇인가요?

    - map은 요소를 갖기 위한 영역만 필요로 하지만 unordered_map은 해쉬 테이블이 필요하기 때문에 별도의 메모리가 추가적으로 필요합니다.

2. 캐시 친화성 측면에서는 어떤 차이가 있나요?

    - map은 노드 기반 구조로 각 노드가 포인터로 연결되어 있어 캐시 미스가 자주 발생합니다. unordered_map은 버킷 배열이 연속 메모리에 있어 상대적으로 캐시 효율이 좋습니다.

## ✅ iterator / 안정성

1. iterator 순회 결과는 두 컨테이너에서 어떻게 다른가요?

    - map은 순차순회, unordered_map은 임의순회입니다.

2. iterator 무효화 규칙은 어떻게 다른가요?

    - map은 erase된 요소의 iterator만 무효화되고, 다른 iterator는 유효합니다. unordered_map은 rehashing이 발생하면 모든 iterator가 무효화됩니다.

## ✅ 실무 / 설계 판단

1. 어떤 경우에 std::map을 사용하는 것이 적절한가요?

    - 삽입, 삭제, 접근이 안정적인 속도를 보장하며 key를 기준으로 정렬된 상태를 기대해야하고 메모리를 아껴야 한다면 map을 사용하는 것이 적절하다고 생각합니다.

2. 어떤 경우에 std::unordered_map을 사용하는 것이 적절한가요?

    - 삽입, 삭제, 접근이 최대한 빠른 속도를 필요로 하고 메모리가 여유로울 경우에 unordered_map을 사용하는 것이 적절하다고 생각합니다.
