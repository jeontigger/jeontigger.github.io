## 📝 모범 답변

### ✅ 기본 개념

1. 컴파일러가 코드를 분석해 성능을 향상시키는 변환 작업입니다.
2. 개발자가 수동으로 최적화하기 어려운 부분을 자동으로 개선하기 위해서입니다.

### ✅ 최적화 레벨 (-O, -O2, -O3)

3. 최적화 강도가 증가하며, -O2가 일반적으로 권장되고 -O3는 공격적 최적화입니다.
4. 디버깅 시 변수와 실행 흐름을 정확히 추적하기 위해서입니다.
5. -O2는 안정적 최적화, -O3는 공격적 최적화로 코드 크기 증가 가능성이 있습니다.

### ✅ 인라인 최적화

6. 함수 호출을 함수 본문으로 치환하는 최적화입니다.
7. 함수 호출 오버헤드 제거와 추가 최적화 기회를 제공합니다.
8. 코드 크기 증가와 컴파일 시간 증가입니다.

### ✅ 링크 타임 최적화 (LTO)

9. 링크 단계에서 여러 오브젝트 파일을 함께 최적화하는 기법입니다.
10. 오브젝트 파일 경계를 넘어 최적화할 수 있기 때문입니다.
11. 링크 시간 증가와 메모리 사용량 증가입니다.

### ✅ 최적화와 디버깅

12. 변수 제거, 코드 재배치, 인라인으로 소스와 실행 코드가 달라지기 때문입니다.
13. -Og 옵션으로 디버깅 가능한 수준의 최적화를 사용합니다.

### ✅ 최적화 트레이드오프

14. 코드 크기 증가, 컴파일 시간 증가, 예상치 못한 동작 변경 가능성 때문입니다.
15. 최적화가 강할수록 코드 크기가 증가할 수 있으나 실행 속도는 향상됩니다.

### ✅ 실무 / 빌드 설정

16. -O2가 성능과 안정성의 균형이 좋아 일반적으로 사용됩니다.
17. 릴리즈는 -O2/-O3, 디버그는 -O0/-Og를 사용합니다.

### ✅ 요약 질문

18. 컴파일러 최적화 옵션은 "코드 성능을 자동으로 향상시키는 컴파일 설정"입니다.
