### ✅ 기본 개념

1. 템플릿 매개변수 선언에서는 동일하지만, 템플릿 내부에서 의존 타입 이름을 명시할 때는 `typename`만 사용 가능합니다.

### ✅ typename이 필요한 경우

1. 템플릿 내부에서 의존 타입 이름을 참조할 때 반드시 필요합니다.
2. 템플릿 매개변수에 의존하는 타입 이름으로, 컴파일 타임에 확정되지 않는 타입입니다.
3. 컴파일러가 타입 이름인지 값인지 구분할 수 없기 때문입니다.
4. 컴파일 에러가 발생하거나 의도하지 않은 해석이 일어날 수 있습니다.

### ✅ typename 생략 가능한 경우

1. 템플릿 매개변수 선언부와 명확한 컨텍스트에서는 생략 가능합니다.
2. 네, 템플릿 매개변수 선언에서는 둘 다 사용 가능합니다.

1. 템플릿 매개변수 선언에서는 둘 다 가능하지만, `typename`이 더 명확한 의미를 전달합니다.
