## 📝 모범 답변

### ✅ 기본 개념

1. 메모리 할당, 이동 의미론, 알고리즘 선택, 컨테이너 선택, iterator 최적화 등이 주요 영역입니다.
2. STL은 자주 사용되므로 작은 최적화도 누적되면 전체 성능에 큰 영향을 미치기 때문입니다.

### ✅ reserve / resize 최적화

1. 재할당을 미리 방지해 불필요한 메모리 할당과 원소 복사/이동을 줄이기 위해서입니다.
2. reserve는 capacity만 증가시키고, resize는 size도 증가시켜 원소를 초기화합니다.
3. 너무 큰 값을 reserve하면 메모리 낭비가 발생할 수 있으므로 적절한 크기를 예측해야 합니다.
4. 삽입할 원소의 개수를 미리 알 수 있을 때 사용합니다.

### ✅ 이동 의미론과 STL

1. 불필요한 복사를 제거해 메모리 할당과 복사 비용을 크게 줄입니다.
2. 이동은 포인터만 전달하므로 복사보다 훨씬 빠르고, 메모리 할당이 필요 없습니다.
3. STL 컨테이너가 이동을 선택할 수 있게 하기 위해서입니다 (예: vector 재할당 시).
4. emplace는 생성 인자를 받아 직접 생성하므로, 이동보다도 더 효율적일 수 있습니다.

### ✅ emplace vs insert

1. emplace는 생성 인자를 받아 직접 생성하고, insert는 완성된 객체를 받아 복사/이동합니다.
2. emplace는 중간 임시 객체 생성과 복사/이동을 건너뛰고 직접 생성하기 때문입니다.
3. 복사/이동 비용이 큰 객체를 삽입할 때, 또는 생성 인자만 있을 때 사용합니다.
4. emplace로 생성된 객체의 생명주기와 예외 안전성을 고려해야 합니다.

### ✅ 알고리즘 선택 최적화

1. 알고리즘 선택에 따라 시간 복잡도와 실제 실행 시간이 크게 달라질 수 있습니다.
2. 컨테이너에 최적화된 멤버 함수가 있으면 그것을 사용하고, 없으면 STL 알고리즘을 사용합니다.
3. random access iterator를 사용하면 더 효율적인 알고리즘을 사용할 수 있습니다.
4. 안정성이 필요하면 stable_sort, 부분 정렬이면 partial_sort, 특정 위치만이면 nth_element를 선택합니다.

### ✅ 메모리 할당 최적화

1. 메모리 할당은 비용이 크고, 빈번한 할당/해제는 단편화를 유발해 성능을 저하시킵니다.
2. vector는 큰 블록을 한 번에, deque는 작은 청크를 여러 번, list는 노드 단위로 할당합니다.
3. reserve/resize로 재할당을 줄이고, 메모리 풀을 사용하며, 적절한 컨테이너를 선택합니다.
4. 특정 메모리 정책이 필요하거나, 메모리 풀을 사용해야 할 때 사용합니다.

### ✅ 컨테이너 선택 최적화

1. 컨테이너 선택에 따라 시간 복잡도, 메모리 사용량, 캐시 효율이 달라집니다.
2. 연속 메모리 컨테이너(vector, deque)를 선택해 캐시 효율을 높입니다.
3. iterator 안정성이 중요하면 list, map, set 등을 선택합니다.
4. 사용 패턴(삽입/삭제/탐색 빈도), 메모리 제약, iterator 안정성 요구사항을 종합적으로 고려합니다.

### ✅ 실무 / 설계 판단

1. 프로파일링으로 병목을 찾고, 영향이 큰 부분부터 최적화합니다.
2. 가독성을 해치지 않는 선에서 최적화하고, 주석으로 최적화 의도를 명확히 합니다.
3. 불가능합니다. 측정 없이 최적화하면 오히려 성능이 나빠질 수 있습니다.
4. STL 성능 최적화는 "프로파일링 기반으로 메모리 할당, 이동 의미론, 알고리즘/컨테이너 선택을 최적화하는 것"입니다.
