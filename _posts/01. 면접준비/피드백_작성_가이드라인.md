# 피드백 작성 가이드라인

## 📋 작업 순서

1. **파일 읽기**: 사용자 답변과 GPT 모범 답변이 포함된 파일을 읽습니다.

2. **질문지 검토**: 
   - 현재 파일에 있는 질문 목록을 확인합니다 (사용자 답변 섹션의 질문들).
   - GPT 답변 섹션의 번호와 사용자 답변 섹션의 질문을 매칭합니다.
   - **사용자 답변 섹션에 없는 질문에 대한 GPT 답변은 반드시 삭제합니다.**
   - 각 카테고리별로 질문 번호를 순차적으로 매칭하여 확인합니다.

3. **피드백 작성**:
   - 사용자 답변과 GPT 모범 답변을 비교합니다.
   - **⚠️ 중요: 정확한 답변은 피드백에 절대 포함하지 않습니다.**
   - **"✅ 정확함" 같은 표현을 사용하지 않습니다.**
   - 오개념(❌), 부정확한 부분(⚠️), 개선이 필요한 부분(💡)만 피드백에 포함합니다.
   - 답변이 누락된 질문은 피드백에 포함합니다.

## ✅ 피드백에 포함할 항목

### 반드시 포함
- ❌ **오개념**: 기술적으로 잘못된 내용
- ⚠️ **부정확한 부분**: 부분적으로 맞지만 정확하지 않은 내용
- 💡 **개선이 필요한 부분**: 정확하지만 더 구체적으로 설명 가능한 내용
- ❌ **답변 누락**: 질문에 대한 답변이 없는 경우

### 포함하지 않음
- ✅ **정확한 답변**: 완전히 맞는 답변은 피드백에 포함하지 않습니다.
  - **"✅ 정확함" 같은 표현을 사용하지 않습니다.**
  - 정확한 답변에 대한 피드백 섹션 자체를 작성하지 않습니다.
  - 정확한 답변은 전체 평가의 "강점"에만 간단히 언급합니다.

## 📝 피드백 형식

```markdown
## 📝 GPT 피드백

### ✅ [카테고리명]

**질문 N: [질문 내용]**
- **현재 답변**: "[사용자 답변]"
- **피드백**:
  - ❌/⚠️/💡 **상태**: [간단한 평가]
  - ✅ **정확한 답변**: "[수정된 답변]" (오개념인 경우)
  - 💡 **보완**: "[개선된 설명]" (부정확하거나 개선 가능한 경우)
  - [추가 설명]

### 📊 전체 평가

**강점:**
- ✅ [정확하게 이해한 부분들 - 간단히 나열]

**개선 필요:**
- ❌ [오개념들]
- ⚠️ [부정확한 부분들]
- [답변 누락 등]

**우선순위:**
1. **높음**: [가장 중요한 오개념이나 누락된 내용]
2. **중간**: [중요한 개선 사항]
3. **낮음**: [부차적인 개선 사항]

**특별 주의사항:**
- [면접에서 자주 나오는 중요한 오개념이나 개념]
```

## 🎯 피드백 작성 예시

### 오개념인 경우
```markdown
**질문 1: stack의 기본 내부 컨테이너**
- **현재 답변**: "stack은 vector로 이루어져있습니다"
- **피드백**:
  - ❌ **오개념**: stack의 기본 내부 컨테이너는 **deque**입니다
  - ✅ **정확한 답변**: "기본적으로 deque를 사용합니다"
  - vector도 사용 가능하지만, 기본값은 deque입니다
```

### 부정확한 부분인 경우
```markdown
**질문 1: iterator category**
- **현재 답변**: "const, random access 등이 존재합니다"
- **피드백**:
  - ⚠️ **부정확**: "const"는 iterator 종류가 아니라 const iterator입니다
  - ✅ **정확한 답변**: "input, output, forward, bidirectional, random access iterator가 있습니다"
  - iterator category는 5가지가 있으며, 계층 구조를 이룹니다
```

### 개선이 필요한 부분인 경우
```markdown
**질문 1: begin()과 end()는 무엇을 반환하나요?**
- **현재 답변**: "begin()은 컨테이너의 논리적 처음 원소를 반환하고..."
- **피드백**:
  - 💡 **보완**: "컨테이너의 시작 iterator와 끝 다음(one-past-the-end) iterator를 반환합니다"
  - "논리적"이라는 표현은 정확하지만, "one-past-the-end"라는 표준 용어를 사용하면 더 좋습니다
  - ⚠️ 주의: "✅ 정확함" 같은 표현은 사용하지 않습니다. 정확한 부분은 피드백에 포함하지 않습니다.
```

### 답변 누락인 경우
```markdown
**질문 1: bidirectional iterator가 필요한 연산**
- **현재 답변**: 답변 누락
- **피드백**:
  - ❌ **답변 필요**: 면접에서 자주 나오는 질문입니다
  - ✅ **정확한 답변**: "역방향 순회(`--`), reverse 알고리즘 등이 bidirectional iterator가 필요합니다"
```

## ⚠️ 주의사항

1. **정확한 답변은 피드백에 포함하지 않습니다. (매우 중요)**
   - "✅ **정확함**: 완벽한 답변입니다" 같은 내용은 절대 포함하지 않습니다.
   - 정확한 답변에 대한 피드백 섹션 자체를 작성하지 않습니다.
   - 정확한 답변은 전체 평가의 "강점"에만 간단히 나열합니다.
   - 오개념이나 개선이 필요한 부분만 피드백에 포함합니다.

2. **없어진 질문의 답변은 반드시 삭제합니다.**
   - 사용자 답변 섹션의 질문 목록을 확인합니다.
   - GPT 답변 섹션의 번호를 사용자 답변 섹션의 질문과 매칭합니다.
   - 사용자 답변 섹션에 없는 질문에 대한 GPT 답변은 반드시 삭제합니다.
   - 각 카테고리별로 질문 번호를 순차적으로 확인하여 매칭합니다.

3. **피드백은 구체적이고 실용적으로 작성합니다.**
   - 단순히 "틀렸다"가 아니라, 왜 틀렸는지, 어떻게 수정해야 하는지 명확히 제시합니다.
   - 면접에서 자주 나오는 주제는 "특별 주의사항"에 명시합니다.

4. **전체 평가는 간결하게 작성합니다.**
   - 강점은 간단히 나열합니다 (정확한 답변에 대한 상세 설명은 제외).
   - 개선 필요 사항은 우선순위를 명확히 합니다.
