## 2. c++ string 고찰

- SSO는 small string optimization으로 짧은 문자열에 대해 힙할당 동적배열을 사용하는 것이 아니라 내부 스택버퍼를 사용하게 됨. 대략 24바이트정도 가능(널 포함)

- string_view는 C++ 17부터 나온 기능인데, 읽기만 하는 string에 대해서 경량해서 받을 수 있는 클래스

- `c_str()`은 `data()`와 크게 다르지 않다. c_str은 null을 마지막에 넣는 것을 반드시 함. c++11 이후부터는 `data()`도 해준다고 함. 그리고 `c_str()`에서 null을 넣어주던 이유가 c 스타일의 문자열에 대응하기 위해서라고 함

## 3. ai 툴에 대한 비교 고찰

- IDE, AI Agent, Model의 구분

- IDE: Visual Studio Code, Cursor, Wrap, Antigravity

- AI Agent: Cursor, Antigravity, OpenCode, Codex, Claude Code

- Model: GPT, Claude Opus, Cluade Sonnet, GLM

### Cursor,Antigravity vs OpenCode, Claude Code

- Cursor와 Antigravity는 IDE와 AI Agent를 결합한 형태

- 그 외는 터미널에서 실행시키는 프로그램의 형태

### 요금

- 우리가 요금 내는 부분은 전부 AI Agent에 내는거라고 생각하면 됨

- IDE는 전부 무료

- Claude Code와 Antigravity는 리필형

- Cursor와 Copilot은 한달 할당제 -> 토큰 개수

- 내 생각엔 리필형이 가장 좋지 않나 싶음

- Antigravity결제시 gemini랑 notebookLM, nano banana 다 쓸 수 있는듯 함

- Codex는 gpt를 같이 쓸 수 있음

### 왜 IDE 결합형말고 다른걸 고려하는가?

- Claude Code 같은 경우에. 생태계가 크고, 다양한 명령어들이 존재. 커스텀 스킬등

- 다양한 명령어들이 GUI로 되어있지 않으면 무조건 터미널 agent를 사용해야 함.

- 다양한 명령어를 쓰지 않을것이라면 IDE 결합형이 훨씬 좋을 수 있음

## 4. Jeff Su의 파일 관리 시스템

[유튜브 영상](https://www.youtube.com/watch?v=pyj49VW6BFg)

- 별건 아니고, 파일을 어떻게 관리하는 것이 좋느냐에 관한 이야기 

1. 모든 파일은 5개의 depth 내로 설계

2. 넘버링 규칙 -> 하나의 depth에서 00부터 99까지 최대 100개만
    - 99번은 Archive 폴더로 활용(필수 x)

3. 모든것은 느슨한 규칙을 적용시키는 것이 좋음
    - 사진이나 동영상같은 경우엔 넘버링대신 날짜로 순서를 보장하는 것이 좋고
    - 무조건 100개 이상 생기는 경우엔 000으로 세자리 넘버링을 하는 것도 좋음
    - 정 정리할 필요가 없는 경우엔 넘버링도 할 필요 없음
    - 규칙에 강박을 갖지 말라는 뜻

4. 추가로, 즐겨찾기를 사용하기를 권장
    - 매일 쓰는거여야 함.
    - 최대 5개만 쓰도록 해야함


## 5. 세컨드 브레인의 저자 '티아고 포르테'의 파라노트 관리법

[유튜브 영상](https://www.youtube.com/watch?v=L-D_xlAdzXI)

- 이건 노트 관리법

- PARA => Project Area Resource Archive

- 실행도가 높은순에서 낮은 순임

- Project: 끝이 명확하고 실행도가 가장 높음. ex) 자격증 합격하기, 시험 공부, 블로그 글 작성, 제품 런칭, 전자책 출간, 이사하기

- 하나의 프로젝트가 완료되면 노트들은 모두 관련된 다른 ARA에 넣게 됨

- Area: 끝이 없음. 진행중. 나의 책임이나 역할이 있음. ex) 운동, 건강, 경제 계획, 직업, 가족 등

- Resource: 프로젝트와 영역에 해당하지 않음. 책임의 유무로 영역과 달라지는데 내가 못해도 상관없으면 자원, 내가 잘해야하면 영역. 관심있는 분야 ex) 음악, 커피, 미술, 역사, 뇌과학, 마케팅, 글쓰기(참고자료, 논문 등이 아님)

- Archive: 실행도 0임. 종료된 프로젝트, 중단된 프로젝트, 책임이 사라진 영역 등이 모임. 

- 노션은 폴더형이 아니라 관계형 데이터베이스 기반이기 때문에 PARA 노트 정리법에 좋음. 여러 프로젝트와 여러 영역, 자원에 속할 수 있음
