# 2. Bootstrap 씬을 만들고 나서 관리하는 법

Bootstrap 씬은 게임이 시작될 때 가장 먼저 실행되는 씬으로, 게임의 초기화 작업을 담당한다.

Unity에디터에서 실행시킬 때 무조건 Bootstrap 씬을 거쳐서 실행되어야 함. 근데 예를들어 타이틀 씬 작업을 하다가 테스트 해보려면 부트스트랩 씬을 다시 로드하고, 그 씬에서 실행시켰다가 다시 타이틀 씬으로 가서 오브젝트를 수정하는 방식이 필요했고, 이게 너무 귀찮았음. 

그래서 다른 방법을 생각함

1. 처음엔 다른 씬에서 부트스트랩씬으로 리다이렉팅하도록 만듬. 근데 이게 현재씬의 오브젝트들에 Awake가 호출되버리고, 씬 전환을 위해 트랜지션씬도 사용하다보니 문제가 좀 생겼었음. 이를 해결하기 보단 2번 방법으로 진행

2. 에디터에서 씬 자체는 Bootstrap 씬을 켜두고, 다른 씬을 수정해야 함. 이 때 프리팹을 사용하면 훨씬 편함. 예를들어, 타이틀 씬의 root 오브젝트를 만들고 해당 오브젝트를 프리팹으로 만듬(캔버스도 마찬가지). 그러면 프리팹을 수정하는 것으로, Bootstrap 씬에서 다른씬을 굳이 방문하지 않아도 수정 가능해짐.

# 3. Vercel을 이용한 배포

Vercel이 Next.js를 개발한 곳 이라고 함

말하고 싶은 것은 2가지. 

1. 프레임워크 설정

어떤 프레임워크를 사용하는지 배포 사이트는 모르기 때문에 프레임워크 설정을 해야 함.

설정하지 않는다면 필요한 사전작업들을 수행하지 않아서 배포해도 정상적으로 작동하지 않음

Vercel 기준으로 Settings에 Build and Deployment에 Framework Settings로 Next.js를 설정하는 작업이 필요했음

2. 환경변수

웹은 환경변수라는게 필요함.

내가 필요했던 것들은 DB 연결 정보, API 키, 구글로그인 등등이 있었음.

로컬에서 테스트 할 때 사용하는 파일이 존재는 하지만, 이것은 엄청 민감한 파일이기 때문에 git에 올리지 않아야 함.

그래서 배포 사이트마다 환경변수를 설정해주는 작업이 필요함.
