# 4. Scriptable Object 사용

프로젝트에서 원래 장비를 생성하는건 전체 랜덤으로 생성되지만, 특정 장비를 생성하는 테스트 시스템이 필요할 것으로 예상함.

장비 클래스는 데이터로만 구성되어있기 때문에, 어떻게 특정 장비를 만드는지에 대한 고민이 필요했음.

그래서 인게임 UI로 옵션들을 드롭박스로 선택하고, 버튼을 누르면 해당 옵션에 맞는 장비를 생성하는 시스템을 먼저 떠올렸음.

그 후 진영이한테 물어보니 Scriptable Object를 사용하면 된다고 함.

생각해보니 예전에 디버깅용 Scriptable Object를 만들었던 적이 있었음. 이걸 이용해서 SO를 통해 장비 데이터를 만드는 시스템으로 특정 장비 생성 시스템을 구현할 수 있었음

# 6. 파이썬 프로그램도 이것저것 만듬

학원 엔진 프레임워크 말고 내가 구현한것들에 대해 대략적인 파일들을 파악하기 위함

1. 커밋 2개를 입력받아 새로 추가된 파일들 찾는 프로그램
2. 1번 프로그램에서 찾은 파일 목록들을 받아 해당 파일들을 하나의 폴더에 새롭게 복사하는 프로그램
3. 2번에서 복사한 파일들을 하나의 txt파일로 합쳐주는 프로그램 - notebookLM에 넣기 위함

이걸 ai에게 시키면서

1. 구현에 대한 개발 가이드를 스탭별로 만들어달라고 요청함.
2. 스탭별로 하나씩 진행하면서 내가 직접 테스트, 후 개선사항을 요청함. 이를 마지막까지 진행함.
3. Readme 작성시킴

이게 프로그램에 대한 나의 이해도, 구현 퀄리티가 높아진다고 느꼈음
